## 2024-04-14 - Optimize Analytics Endpoints with DB-Level Aggregation & Shared Prisma Client
**Learning:** Analytics endpoints in this codebase often overfetch large datasets by using `findMany` into Node.js memory instead of utilizing the database's aggregation capabilities (e.g. `groupBy`). Additionally, there is a recurring anti-pattern where multiple route handlers instantiate their own `PrismaClient` using `new PrismaClient()`, which can lead to database connection exhaustion and memory bloat.
**Action:** When working on API routes (especially for analytics), use Prisma's native `.groupBy()`, `.aggregate()`, or `.count()` methods to push calculations to the database layer. Always import the shared, singleton Prisma client from `@/lib/prisma` rather than creating a new instance.
