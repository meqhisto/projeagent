## 2024-04-18 - Overfetching in Analytics APIs and Instantiating PrismaClient
**Learning:** Analytics endpoints (`app/api/analytics/monthly-trend/route.ts`, `app/api/analytics/pipeline/route.ts`, `app/api/analytics/kpis/route.ts`) are currently fetching all records from the database using `.findMany()` and performing aggregate calculations (counts) in Node.js memory. Also, they are instantiating a new `PrismaClient` instead of using the shared singleton from `lib/prisma.ts`. This causes high memory usage, long query times, and database connection exhaustion.
**Action:** Replace `new PrismaClient()` with `import { prisma } from "@/lib/prisma";`. Use Prisma's aggregation features like `.groupBy()`, `.count()`, and `.aggregate()` to push these calculations to the database instead of fetching records into memory. Use `Promise.all` to perform multiple aggregations concurrently.
