## 2024-05-07 - Overfetching full relations for just counts in Prisma
**Learning:** In Prisma, using `include: { matches: true, ratings: true }` when only the count and average calculation is needed leads to massive overfetching, N+1 query patterns, and increased JSON payload size on the backend API route. Prisma's `_count` parameter inside `include` provides a native, optimized way to retrieve relation counts without fetching the actual rows.
**Action:** When a UI list only needs to show the number of related items or calculate an aggregate (and the frontend is just accessing `.length`), modify the backend Prisma `include` to fetch `_count` or calculate averages on the database side (or fetch specific fields and use _count), and update the frontend to use `item._count.field` instead of `item.field.length`.
