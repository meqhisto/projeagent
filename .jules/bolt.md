## 2024-05-18 - Replacing `findMany` with `groupBy` for KPIs and Analytics
**Learning:** Using `findMany` to fetch all database records into Node.js memory just to calculate aggregate counts or sums is a performance anti-pattern. This codebase originally performed O(n) array filtering and mapping on the entire `parcel` dataset to generate the CRM pipeline stages summary. As the dataset grows, this approach drastically increases database payload size and Node.js memory footprint.
**Action:** Instead, leverage Prisma's `.groupBy()` method with `_count: { _all: true }` (or `.aggregate()`) to push processing to the database layer. This ensures only the aggregated results are returned to the application, maintaining constant memory overhead regardless of database size. Note: Always verify how `null` or missing values are handled in the DB results (e.g., mapping `null` `crmStage` to a default value like `"NEW_LEAD"`).
