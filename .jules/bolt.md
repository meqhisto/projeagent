## 2025-05-19 - API List Endpoint _count optimization
**Learning:** Over-fetching entire related objects (like `matches: { include: { parcel: true, customer: true } }` and `ratings: true`) just to get their counts is a common N+1/memory-bloat pattern in Prisma. We can use Prisma's `_count` aggregate inside the `include` block. Furthermore, average ratings can be pre-calculated via `_avg` or computed in-memory based on fetched data, but pulling ALL nested data (like `parcel` and `customer` objects within `matches` just to get `matches.length` in the UI) creates unnecessary large payloads.
**Action:** When an endpoint returns a list of items and the UI only needs the count of related records, use Prisma's `include: { _count: { select: { relation: true } } }` instead of fetching full relational arrays.
