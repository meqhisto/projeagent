## 2024-11-20 - [Performance] Prevent N+1 and Overfetching using _count and omit
**Learning:** Returning fully nested relation objects (e.g., `matches` with its nested `parcel` and `customer` info, and all fields of `ratings`) in a GET list endpoint when the UI only needs relation counts and a simple aggregate (like `averageScore`) creates enormous memory overhead and inflates payload size unnecessarily.
**Action:** Use Prisma's `_count` parameter inside the `include` block for relation lengths. When aggregating data in Node.js (e.g. calculating an average), select ONLY the minimum numeric fields needed. MOST IMPORTANTLY, remember to omit the intermediary fetch data from the mapped result using object destructuring (`const { ratings, ...rest } = item;`) so it doesn't leak into the returned JSON payload.
