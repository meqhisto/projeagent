## 2025-04-20 - Avoid Memory Overfetching with Concurrent Prisma Aggregations
**Learning:** In analytics routes (like `app/api/analytics/kpis/route.ts`), using `findMany` to fetch large sets of records (especially with `include` statements for relations like `zoning`) and then performing `.filter` and `.reduce` operations in Node.js leads to severe memory exhaustion, high database data transfer latency, and unhandled promises. Furthermore, instantiating `new PrismaClient()` in every API route call can easily exhaust connection pools in a serverless environment.
**Action:** Replace `findMany` + in-memory filtering with concurrent database-level aggregations using `Promise.all` alongside `prisma.count` and `prisma.aggregate` to push computation to PostgreSQL. Always import and use the shared singleton `prisma` instance from `@/lib/prisma`.
