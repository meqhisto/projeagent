## 2024-03-24 - Prisma Include Over-fetching
**Learning:** The codebase previously fetched all related image records in list endpoints (like `/api/parcels` and `/api/properties`) using `images: true` in Prisma `include`. This led to over-fetching since the UI only uses a single thumbnail. However, when limiting fetched fields with `select`, it's critical to include relational identifiers (like `id`) as they are often required by frontend caching and list rendering keys. It is also important not to truncate related records in `POST` endpoints, as the frontend uses the full response to hydrate state.
**Action:** Always use `select`, `orderBy`, and `take` inside Prisma `include` blocks when fetching lists of items that have related images to limit payload size. Always include the `id` field in the `select`. Only apply `take: 1` to `GET` list endpoints, not to mutations (`POST`, `PATCH`), to preserve correct state hydration.
