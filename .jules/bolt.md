## 2024-06-07 - Database level aggregations instead of in-memory maps
**Learning:** In the `app/api/properties/stats/route.ts` API route, all properties, related units and all transactions for the year are fetched into node's memory to perform calculations for statistics. This creates unnecessary memory usage and processing overhead in Node.js instead of leveraging database aggregation capabilities using `prisma.property.aggregate`, `prisma.unit.aggregate`, and `prisma.transaction.aggregate`.
**Action:** Replace full array fetches (like `prisma.property.findMany`) with database-level aggregations (`aggregate`, `count`, `groupBy`) where possible to reduce network transfer and memory overhead in API endpoints.
