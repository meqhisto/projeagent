## 2024-04-02 - [Pushing Aggregations to Database Layer for Analytical Endpoints]
**Learning:** Found multiple instances where analytical endpoints (e.g., `app/api/analytics/pipeline/route.ts`) were fetching the entire `parcel` dataset using `findMany` just to perform counting and grouping in Node.js memory. This scaling is O(N) both in terms of database transfer time and Node.js memory allocation. Additionally, there were unsafe usages of `new PrismaClient()` causing connection pool leaks.
**Action:** Always prefer pushing grouping logic to the database by using `prisma.model.groupBy` and `prisma.model.count` along with mapping the results in memory using Maps to accumulate fallback logic (e.g., tracking `NULL` stages back to defaults). Also, always ensure the shared singleton client is used.
